# McKinsey Visual Identity Reference

Apply these standards to every slide generated by mckinsey-slide.

---

## Brand Palette

| Name              | Hex       | Usage |
|-------------------|-----------|-------|
| McKinsey Blue     | `#0E4D92` | Primary accent, action title emphasis, chart primary, header underlines |
| McKinsey Dark     | `#1A1A1A` | Body text, titles, borders — near-black for maximum contrast |
| McKinsey Mid Grey | `#6B6B6B` | Secondary text, axis labels, footnotes |
| McKinsey Light Grey| `#E8E8E8` | Alternating table rows, panel backgrounds |
| White             | `#FFFFFF` | Slide backgrounds, text on dark elements |
| McKinsey Red      | `#C0392B` | Losses in waterfalls, risk flags, negative indicators |
| McKinsey Green    | `#1E8449` | Gains in waterfalls, positive indicators |
| McKinsey Amber    | `#D68910` | Warnings, medium-risk items, selective items in 3x3 matrix |

## Slide Layout

### Dimensions
- **Slide size:** Widescreen 16:9 — 25.4cm × 14.29cm

### Standard Zones
```
┌─────────────────────────────────────────────────┐
│ [McKinsey Blue underline — 2pt, full width]     │
│                                                 │
│  TITLE ZONE (y=0.3–1.0in)                       │
│  Full-width, 18-20pt bold, #1A1A1A              │
│  [2pt McKinsey Blue underline beneath title]    │
│                                                 │
│  CONTENT ZONE (y=1.1–5.0in)                     │
│  Varies by framework — see frameworks.md        │
│                                                 │
│─────────────────────────────────────────────────│
│ [Source left, 8pt | Page right, 8pt]           │
└─────────────────────────────────────────────────┘
```

**Key difference from EY/BCG:** McKinsey slides use a thin blue underline beneath the action title, not a coloured header bar. White backgrounds everywhere — maximally clean.

### Cover Slide Layout
- White background (clean, no coloured panels)
- Deck title: 28pt Bold, #1A1A1A
- McKinsey Blue 3pt horizontal rule below title
- Subtitle / date / client: 16pt Regular, #6B6B6B
- Firm name/logo: bottom-right, McKinsey Blue

### Section Divider Layout
- White background
- Section number: Large (48pt), McKinsey Blue, left-aligned
- Section title: 24pt Bold, #1A1A1A, below number
- 2pt McKinsey Blue rule across full width below title

## Visual Style Rules

### General
- **Minimal design:** No coloured backgrounds on content slides (only cover + dividers)
- No drop shadows, no gradients, no 3D effects
- No decorative elements — every visual element must carry information
- Consistent left-margin alignment: 0.5in

### Tables
- No coloured header rows — use **bold** text and a bottom border (2pt #1A1A1A) instead
- Alternating rows: white / Light Grey (#E8E8E8)
- Column borders: 0.5pt Light Grey
- Outer border: 1pt #1A1A1A
- Highlight rows: McKinsey Blue left border (3pt)

### Charts
- Primary bars/lines: McKinsey Blue
- Secondary series: McKinsey Mid Grey
- Positive waterfall bars: McKinsey Green
- Negative waterfall bars: McKinsey Red
- Selective/hold bars: McKinsey Amber
- All charts: white background, no chart border
- Gridlines: very light grey (0.5pt) horizontal only, no vertical
- Data labels: #1A1A1A, 10pt, outside end

### Annotations
- Key insight: McKinsey Blue left border (3pt), no background fill
- Warning: Amber border (3pt)
- Critical issue: Red border (3pt)
- Source citation: 8pt, #6B6B6B, bottom-left of chart
- Waterfall connectors ("→ therefore / but / so"): italic, 10pt, McKinsey Mid Grey

## pptxgenjs Bootstrap

Use this boilerplate for every generated `slides.ts`:

```typescript
import pptxgen from 'pptxgenjs';

const pptx = new pptxgen();

// Slide dimensions (widescreen 16:9)
pptx.defineLayout({ name: 'WIDESCREEN', width: 10, height: 5.63 });
pptx.layout = 'WIDESCREEN';

// McKinsey brand colours
const MK = {
  blue:      '0E4D92',
  dark:      '1A1A1A',
  midGrey:   '6B6B6B',
  lightGrey: 'E8E8E8',
  red:       'C0392B',
  green:     '1E8449',
  amber:     'D68910',
};

// Slide master helper (McKinsey minimalist — blue title underline, no top bar)
function addSlideChrome(slide: pptxgen.Slide, title: string, pageNum: number, total: number) {
  // Title
  slide.addText(title, {
    x: 0.5, y: 0.22, w: 9, h: 0.65,
    fontSize: 20, bold: true, color: MK.dark, fontFace: 'Calibri',
  });
  // McKinsey Blue underline beneath title
  slide.addShape(pptx.ShapeType.rect, { x: 0.5, y: 0.9, w: 9, h: 0.03, fill: { color: MK.blue } });
  // Page number
  slide.addText(`${pageNum} / ${total}`, {
    x: 8.5, y: 5.43, w: 1.4, h: 0.2, fontSize: 8, align: 'right', color: MK.midGrey,
  });
}
```

## Markdown Output Style (mckinsey-slide specific)

Since mckinsey-slide outputs structured markdown (not .pptx), apply these conventions:

```markdown
### Slide N: [ACTION TITLE — declarative insight sentence]

**Framework:** [Pyramid Principle / MECE Tree / 7S / Three Horizons / GE-McKinsey / Value Bridge / Action Plan]

**Pyramid position:** [Governing Thought / Argument N / Evidence under Argument N]

**Bullets** (3-5 max, 7 words max each, MECE):
- [Insight — quantified where possible]
- [Insight]
- [Insight]

**Visual Proof Point:**
- Chart type: [waterfall / 2x2 / 3x3 matrix / three horizons / scatter / comparison table]
- Data: [what it shows and what it proves]

**So What?:** [One sentence — what must the audience conclude?]

**→ [therefore / but / so]:** [Next slide title]

**Speaker notes:** [1-2 sentences said aloud, not on slide]
```

## Slide Quality Checklist

Before finalising any slide:
- [ ] Title is a declarative insight sentence ("Revenue declined 23% due to SMB churn") — not a topic label ("Revenue")
- [ ] "So What?" can be answered in one sentence
- [ ] Slide passes the Cover Test (story readable from titles alone)
- [ ] MECE check: bullets have no overlap, no gaps
- [ ] Maximum 5 bullets; maximum 7 words per bullet
- [ ] Pyramid position annotated (Governing Thought / Argument / Evidence)
- [ ] Action title underlined with McKinsey Blue rule (in markdown: bold + "---" separator)
- [ ] Source cited if any external data is used
- [ ] Waterfall connector to next slide present
