Apply these standards to every slide generated by bcg-slide.

## Slide Layout

### Dimensions
- **Slide size:** Widescreen 16:9 — 25.4cm × 14.29cm

### Standard Zones
```
┌─────────────────────────────────────────────────┐
│ [BCG DARK NAVY BAR — 0.3in] logo top-right      │
│─────────────────────────────────────────────────│
│                                                 │
│  TITLE ZONE (y=0.4–1.1in)                       │
│  Full-width, 20pt bold, #003B5C                 │
│                                                 │
│  CONTENT ZONE (y=1.2–5.0in)                     │
│  Varies by framework — see frameworks.md        │
│                                                 │
│─────────────────────────────────────────────────│
│ [FOOTER — source left | page number right, 8pt]│
└─────────────────────────────────────────────────┘
```

### Cover Slide Layout
- Background: BCG Dark Navy (`#003B5C`) full slide
- Deck title: 28pt Bold, white text
- Subtitle / date / client: 16pt Regular, BCG Light Blue
- BCG wordmark: top-right, white
- Confidentiality notice: bottom-left, 8pt, Mid Grey

### Section Divider Layout
- Left 40%: BCG Dark Navy fill
- Right 60%: White
- Section number: Large display (60pt), BCG Light Blue, right panel
- Section name: 20pt Bold, #003B5C, right panel below number

## Visual Style Rules

### General
- White backgrounds for all content slides (no coloured backgrounds except cover + dividers)
- Clean, minimal layout — no drop shadows, no gradients, no 3D effects
- Grid alignment: all elements snap to 0.1-inch grid
- Maximum 3-5 bullets per slide; one key message per slide

### Tables
- Header row: BCG Dark Navy background, white bold text
- Alternating body rows: white / Light Grey (#F0F2F3)
- 1pt Dark Grey border on all cells
- Highlight column for recommended option: BCG Light Blue header

### Charts
- Primary bars/lines: BCG Dark Navy
- Secondary series: BCG Light Blue
- Positive waterfall bars: BCG Green
- Negative waterfall bars: BCG Red
- All charts: white background, no chart border, no gridlines (or very light grey 0.5pt)
- Data labels: #3C4A56, 10pt

### Annotations
- Key insight callout: BCG Light Blue left border (3pt), light grey background panel
- Warning callout: BCG Amber left border (3pt)
- Critical finding callout: BCG Red left border (3pt)
- "Therefore / But / So" connectors: italicised, BCG Mid Grey, bottom of slide

## pptxgenjs Bootstrap

Use this boilerplate for every generated `slides.ts`:

```typescript
import pptxgen from 'pptxgenjs';

const pptx = new pptxgen();

// Slide dimensions (widescreen 16:9)
pptx.defineLayout({ name: 'WIDESCREEN', width: 10, height: 5.63 });
pptx.layout = 'WIDESCREEN';

// BCG brand colours
const BCG = {
  navy:      '003B5C',
  green:     '009A44',
  lightBlue: '4DA1C8',
  darkGrey:  '3C4A56',
  midGrey:   '9EA8B3',
  lightGrey: 'F0F2F3',
  red:       'C0392B',
  amber:     'E67E22',
};

// Slide master helper
function addSlideChrome(slide: pptxgen.Slide, title: string, pageNum: number, total: number) {
  // BCG Dark Navy top bar
  slide.addShape(pptx.ShapeType.rect, { x: 0, y: 0, w: 10, h: 0.22, fill: { color: BCG.navy } });
  // Title
  slide.addText(title, {
    x: 0.5, y: 0.32, w: 8.5, h: 0.7,
    fontSize: 20, bold: true, color: BCG.navy, fontFace: 'Calibri',
  });
  // Footer bar
  slide.addShape(pptx.ShapeType.rect, { x: 0, y: 5.43, w: 10, h: 0.2, fill: { color: BCG.lightGrey } });
  // Page number
  slide.addText(`${pageNum} / ${total}`, {
    x: 8.5, y: 5.43, w: 1.4, h: 0.2, fontSize: 8, align: 'right', color: BCG.midGrey,
  });
}

console.log('✓ bcg-deck.pptx written');
```

## Markdown Output Style (bcg-slide specific)

Since bcg-slide outputs structured markdown (not .pptx), apply these conventions:

```markdown
### Slide N: [ACTION TITLE IN BOLD CAPS IF INSIGHT]

**BCG Framework:** [type]

**Issue tree branch:** [which branch of the MECE tree this supports]

**Bullets:**
- [Insight — specific, declarative, evidence-backed]
- [Insight]
- [Insight]

**Visual Proof Point:**
- Chart type: [waterfall / 2x2 / three horizons / experience curve / comparison table]
- Data: [what it shows and why it matters for the hypothesis]

**→ [therefore / but / so]:** [Next slide title — waterfall connector]

**Speaker notes:** [1-2 sentences said aloud, not shown on slide]
```

## Slide Quality Checklist

Before finalising any slide:
- [ ] Action title is a declarative insight sentence (not a topic label)
- [ ] Slide connects to the next via "therefore", "but", or "so"
- [ ] MECE check: bullets have no overlap and no gaps
- [ ] Issue tree branch annotated
- [ ] Maximum 5 bullets
- [ ] "So what?" can be answered for this slide
- [ ] Visual proof point specified (even if rendered as text in markdown)
- [ ] Speaker notes included (context not on the slide itself)
